PayShield AI Frontend UI (Full Application)


### STEP 0 — SETUP AND REPO STRUCTURE
Generate a complete, production-grade frontend for PayShield AI, a real-time digital payment fraud detection platform. Use React 18+ with TypeScript. The UI must match the provided designs exactly, with dark/light themes (toggleable), responsive layout (mobile-first), and seamless integration with the backend (FastAPI APIs). Focus on analyst workflows, real-time updates, and enterprise-grade dashboards.

Start by initializing the project:
  - Run `npm create vite@latest payshield-frontend -- --template react-ts` in a new directory. This sets up a React + TypeScript project.
  - Choose "React" as the framework and "TypeScript" variant if prompted.
  - Navigate into the directory: `cd payshield-frontend`.
  - Install base dependencies: `npm install`.

- **Install Additional Dependencies**:
  - Run `npm install tailwindcss @tailwindcss/vite postcss autoprefixer @types/react @types/react-dom axios recharts @mui/material @mui/icons-material socket.io-client react-router-dom @types/socket.io-client @types/react-router-dom`.

- **Configure Tailwind**:
  - Run `npx tailwindcss init -p` to generate `tailwind.config.js` and `postcss.config.js`.
  - Update `tailwind.config.js` with content paths and custom theme:
    ```javascript
    /** @type {import('tailwindcss').Config} */
    export default {
      content: [
        "./index.html",
        "./src/**/*.{js,ts,jsx,tsx}",
      ],
      theme: {
        extend: {
          colors: {
            primary: '#0EA5E9', // cyan
            accent: '#F59E0B',  // orange
            error: '#EF4444',   // red
            success: '#10B981', // green
          },
          fontFamily: {
            sans: ['Inter', 'sans-serif'],
          },
        },
      },
      plugins: [],
    };
    ```
  - Update `src/index.css` (or create if missing) to include Tailwind directives:
    ```css
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    ```
  - In `vite.config.ts`, ensure it's set up for React (Vite does this by default with the template).

- **Set Up ESLint and Prettier for Code Quality**:
  - Install: `npm install --save-dev eslint prettier eslint-plugin-react eslint-plugin-react-hooks @typescript-eslint/eslint-plugin @typescript-eslint/parser eslint-config-prettier eslint-plugin-prettier`.
  - Create `.eslintrc.cjs` (or `.eslintrc.json`):
    ```javascript
    module.exports = {
      env: { browser: true, es2021: true, node: true },
      extends: [
        'eslint:recommended',
        'plugin:react/recommended',
        'plugin:react-hooks/recommended',
        'plugin:@typescript-eslint/recommended',
        'prettier',
      ],
      parser: '@typescript-eslint/parser',
      parserOptions: { ecmaFeatures: { jsx: true }, ecmaVersion: 'latest', sourceType: 'module' },
      plugins: ['react', '@typescript-eslint', 'prettier'],
      rules: {
        'prettier/prettier': 'error',
        'react/react-in-jsx-scope': 'off', // Not needed with React 17+
      },
      settings: { react: { version: 'detect' } },
    };
    ```
  - Create `.prettierrc`:
    ```json
    {
      "singleQuote": true,
      "semi": true,
      "trailingComma": "es5",
      "printWidth": 100
    }
    ```
  - Add scripts to `package.json`:
    ```json
    "scripts": {
      "lint": "eslint src --ext ts,tsx --report-unused-disable-directives --max-warnings 0",
      "format": "prettier --write src/**/*.{ts,tsx,css}"
    }
    ```
  - Run `npm run lint` and `npm run format` to test.

This Vite-aligned setup maintains compatibility with the original compendium structure (e.g., src/app, pages, components). Run `npm run dev` to start (faster than CRA's `npm start`). If integrating with backend, update `vite.config.ts` for proxying:
```typescript
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/v1': 'http://localhost:8000', // Proxy API calls to backend
    },
  },
});

- Use React Router for routing (e.g., /landing, /login, /dashboard, protected routes with auth).
- Implement theme toggle (dark: bg-gray-900 text-white; light: bg-white text-gray-900).
- For backend integration: All data fetches use apiClient (e.g., get('/analytics') for charts). Handle loading/error states. Use sockets for real-time (e.g., new alerts trigger toast). Ensure JWT auth: Store token in localStorage, add to headers. No issues later: Use types for API responses, mock backend in dev.

### GLOBAL DESIGN THEME
- Overall: Modern, enterprise fintech UI. Dark mode default (bg-gray-900, text-gray-200); light mode toggle (bg-white, text-gray-800). Rounded corners (border-radius 8px), shadows for cards, cyan accents (#0EA5E9).
- Typography: Inter font, headings bold (h1 2xl, h2 xl), body md.
- Colors: Primary cyan #0EA5E9, success green #10B981, warning orange #F59E0B, error red #EF4444, neutral gray #4B5563.
- Icons: MUI icons, flags for countries.
- Responsiveness: Grid/flex for layouts, mobile: stack vertically.
- Real-time: Use Socket.io for live updates (e.g., activity feeds, alerts).
- Accessibility: ARIA labels, keyboard nav.
- No extra features; use mock JSON for data until backend connect.
